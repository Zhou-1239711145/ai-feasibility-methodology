# 大模型可行性评估方法论（完整版）

## 核心问题
如何判断一项任务是否适合用大模型？

## 四个评估维度

| 维度 | 要回答的问题 |
|------|-------------|
| 组合数 | 业务有多少种场景组合？ |
| 边际收益 | 调Prompt能提升多少？ |
| 失败收敛 | 错误是随机分布还是集中在某类场景？ |
| 业务风险 | 判断错了会赔钱吗？ |

## 评估流程（6步法）

1. 梳理业务组合数（1-2天）
2. 设计核心用例（1天）
3. 建立人工基线（2-3天）
4. 多轮Prompt调优（1-2天）
5. 分析边际收益与失败模式（0.5天）
6. 给出结论（0.5天）

**总周期：约1周**

## 结论判断标准

| 结论 | 条件 |
|------|------|
| 可用 | 通过率 > 95% + 失败收敛 |
| 部分可用 | 通过率 80%-95% + 有明确边界 |
| 不可用 | 通过率 < 80% 或 失败不收敛 |

## 回头票案例验证

- 业务组合数：1000+种
- 4轮Prompt调优：60% → 81.2% → 84.4% → 84.4%
- 边际收益归零点：第3轮后
- 失败模式：不收敛
- 结论：不可用，建议规则引擎
